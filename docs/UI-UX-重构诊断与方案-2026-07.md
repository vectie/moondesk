# MoonDesk UI/UX 重构：诊断报告与改进方案

> 日期：2026-07-21
> 范围：`ui/rabbita-desk` 前端（MoonBit + Rabbita），覆盖 Packs / 桌面 / 知识库 / 代码 / Flow 五个模式
> 验证：284/284 白盒测试通过；本地 host 全页面截图走查

---

## 一、总诊断：为什么"不知道做什么"

走查全部五个模式后，核心问题不是"丑"，而是**信息架构没有回答三个问题**：我在哪、这里最重要的东西是什么、下一步点哪里。具体表现为：

1. **每个页面都在同时展示"所有信息"**。桌面模式左侧栏同时塞着 MoonGate 卡片、存储诊断、17 个产品清单；Flow 空态同时出现 hero、画布占位、空运行卡、四张跨产品跳转卡。没有视觉层级，用户无法形成焦点。
2. **文案中英混排**。顶栏 Packs/桌面/知识库/代码/Flow 与正文 "What do you want to run?" 混杂。根因：i18n 采用运行时文本替换（`moonsuite-i18n.js` + `locales/*.json` 词典），词典只覆盖 88 个键，大量新字符串未被翻译。
3. **五个模式的布局范式互不统一**：桌面是三栏文件管理器，知识库是"轨道+面板+预览"三栏，Flow 是文档流，代码是两栏。切换模式的学习成本被放大。
4. **状态文案泄漏内部错误**。代码页底部常驻 "Invalid archived MoonCode session listing"——一个非关键接口的错误被直接渲染给用户。

## 二、知识库（Wiki）：抓不住重点 & 不知道怎么编辑

### 发现的问题

| # | 问题 | 根因 |
|---|------|------|
| W1 | 左侧"轨道 + 面板"双重嵌套：轨道（资料库/搜索/收件箱/活动/审核/More）+ 面板（Current book 卡片、书列表、Overview/Pages/Inbox/Published 四个页签） | 当前书在卡片和列表里重复出现两次；Overview 页签默认激活却显示 "Nothing here yet / Create first note"——书里明明有页面 |
| W2 | "新建笔记"入口藏在收件箱流程里，页面上没有任何显眼的创作入口 | 创建动线没有被显式设计 |
| W3 | **脏检查误报（真 bug）**：进入编辑模式后一个字不改，切走也弹 "Discard unsaved changes?" | `markdown_navigation_needs_confirmation` 只判断"编辑器是否激活"，从未比较内容与原文 |
| W4 | 一进入编辑模式，状态栏立刻显示 "Unsaved changes" | `wiki_editor_status_label` 对 `editing` 状态无条件返回该文案 |
| W5 | `[[wiki link]]` 不渲染为链接 | 渲染管线未处理双链语法（本轮未改，见"遗留事项"） |

### 改进方案（已实施）

- **面板重构**：删掉重复的 Current book 大卡；顶部是紧凑的书切换器（仅列表）；其下是贯穿宽度的主按钮 **"+ 新建笔记"**；再下是五个语义清晰的页签：主页 / 页面 / 素材 / 收件箱 / 已发布。
- **新增"主页"态**：当未进入任何目录时，面板显示四张导航卡（页面/素材/收件箱/已发布，各带一句人话说明），替代原来误导性的 Overview 空态。
- **脏检查修复**：Model 新增 `markdown_editor_original`，进入编辑时快照原文；导航拦截与状态文案都改用真实的内容比对（`markdown_editor_dirty`）。未修改时切走**不再弹窗**；状态栏区分 **"编辑中" / "未保存的修改" / "已保存"**。对应测试夹具已更新并补充"无修改"断言。

涉及文件：`moonwiki_workspace_views.mbt`、`app_workspace_state.mbt`、`app_update_navigation_actions.mbt`、`app_update_workspace_surface_actions.mbt`、`moonwiki_user_projection.mbt`、`app_wiki_ux_wbtest.mbt`、`app_model.mbt`。

## 三、Flow：从"文档页"到真正的无限画布

### 发现的问题

| # | 问题 | 根因 |
|---|------|------|
| F1 | **完全不是画布**：有 Work graph 时只是按深度分列的纵向滚动布局；没有平移、没有自由缩放、不能拖节点 | 视图用普通文档流实现，缩放只有 75/100/125 三档 CSS class |
| F2 | 空态时整个模式退化成提示卡片 + 大 hero + 四张跳转卡 | 空态没有独立的画布表达 |
| F3 | 17 个产品的调色板全量平铺，而图里只有 6 个产品 | 列表未按"图中实际用到"过滤 |
| F4 | 顶部 hero 占掉近 1/4 屏 | 装饰性文案与功能无关 |

### 改进方案（已实施）

**交互模型**（新增 `flow_canvas_actions.mbt`）：

- Model 新增 `flow_pan_x/flow_pan_y`（屏幕像素）、`flow_zoom`（0.25–2.5 连续值）、`flow_pan_dragging`、`flow_drag_node_id`、`flow_node_positions`（用户拖动后的节点坐标覆盖表）。
- 消息：`FlowCanvasWheel` / `FlowPanStart|Move|End` / `FlowNodeDragStart` / `FlowCanvasReset`。
- **平移**：在空白处按住拖动；**缩放**：滚轮连续缩放（步进 ×1.12，锚定内容中心，画面不跳）；**拖节点**：按住节点左上角手柄 ⠿ 自由摆放，坐标按当前缩放换算回画布坐标系。
- 节点拖拽与画布平移共用指针流：节点手柄的 mousedown 先置 `flow_drag_node_id`，冒泡到视口的平移起点判定到该标记后自动放弃平移——无需 stopPropagation。

**视图层**（重写 `moonflow_canvas_views.mbt`）：

- 视口（`overflow:hidden` + 抓手光标）→ 4800×3200 点阵表面（`transform: translate + scale`，原点 0 0）→ SVG 贝塞尔连线层（激活/未激活两种描边）→ 绝对定位节点卡。
- 顶部紧凑工具条：标题 + 状态、− / 缩放读数 / +、重置视图、保存组合。
- 产品调色板改为**可折叠浮动面板**，且只列出图中声明的产品（6 个而非 17 个）。
- **空态也是画布**：同样的可平移缩放点阵空间，中央一张引导卡（"还没有可执行的 Work 图" + 打开可执行文档按钮），不再是提示碎片。
- 底部常驻操作提示条："拖拽空白处平移 · 滚轮缩放 · 拖动节点手柄重新排列"。
- 页面 hero 压缩为单行标题条。

**已知取舍**：滚轮缩放的锚点是"内容中心"而非光标（Rabbita 侧拿不到视口矩形，无法做光标锚定），缩放大图形时手感略异于 Figma；节点自由位置仅存于前端会话，不落盘。两者都已记录为后续迭代项。

## 四、Packs：独立 Tab 确认 + 信息降噪

- 现状确认：Pack 系统**已经是独立 Tab**（顶栏第一位），本轮保留该信息架构并打磨。
- 删除了首屏常驻的"Pack→MoonDesk→MoonFlow→MoonClaw"四段架构说明条与底部说明段落，合并为一条可折叠的 "Pack 如何运行"。
- hero 副标题从 38 词压缩为一句："Pack 即应用：打开一个进入领域工作，或在 Flow 中查看其组合。"
- 修复翻译串扰："{count} installed" 模板误匹配了状态文案 "No domain packs installed"（渲染出"已安装 No domain packs 个"），补充精确匹配键后解决。

## 五、代码（Code）：轻改

- 吞掉非关键错误：归档会话列表接口失败时不再把 "Invalid archived MoonCode session listing" 渲染到页面上（该接口在当前 host 上不存在，属于可选能力）。
- 补齐中文文案：MoonClaw 初始化横幅、询问输入框、会话计数等 10 个键。
- 布局维持不变——你说得对，它是五个模式里信息架构最清晰的。

## 六、i18n 机制与本轮词条

词典 `locales/en-US.json` / `zh-Hans.json` 从 88 键扩到 149 键，覆盖本轮触及的所有界面文案；用 moonlib 的 `generate-web-i18n.mjs` 重新生成 `public/moonsuite-i18n.js`。机制要点：文本节点**精确匹配优先**，其次 `{name}` 模板匹配——新增模板键时必须检查它不会误吞更长的状态文案（F/四 的 bug 即此因）。

## 七、遗留事项（建议下一迭代）

1. **Wiki 双链渲染**：`[[页面]]` 语法在预览中渲染为可点击链接。
2. **Flow 光标锚定缩放**：需要 Rabbita 侧暴露元素 bounding rect（或一小段 JS FFI），把缩放锚点从内容中心换到光标。
3. **Flow 节点位置持久化**：拖动后的坐标可随 composition 一起存回 `flow/work-graph.json`（需要 host 端 schema 支持）。
4. **Wiki 编辑器工具栏**：加粗/链接/列表快捷键与 Cmd+S 保存。
5. **桌面模式**：信息密度第二高的页面（诊断卡片、17 产品清单建议折叠），本轮未动，建议沿用 Flow 的"可折叠浮动面板"范式处理。
