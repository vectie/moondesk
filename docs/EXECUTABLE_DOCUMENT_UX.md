# Executable document UX

MoonDesk treats a MoonBook as an executable document when the user can trace a
statement of intent all the way through governed work—not merely edit prose and
launch an unrelated job.

Flow mode therefore exposes, for every run and item:

- MoonBook, declaration revision, and source identity;
- declaration identity, product owner, operation, dependencies, and requested authority;
- acceptance criteria, attempts, blockers, and the reason for the current state;
- input/output identities, evidence artifacts, approval, and review authority.

The primary UX rule is “explain the state.” A badge alone is insufficient.
Ready items say why they are executable, review items say what remains, accepted
items identify the evidence/review basis, and failures retain actionable causes.

Flow is orchestration and inspection. Wiki remains document authoring; Code
remains implementation. Crossing modes must preserve the selected MoonBook and
the same workspace-root authority boundary.
